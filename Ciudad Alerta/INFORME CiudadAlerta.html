2. Backend Core

2.1 Servidor Express (server.js)

  • Express con middleware: helmet, compression, cors, rate-limit
  • JWT authentication (24h expiration)
  • Multer para uploads (5MB max, imagenes jpg/png/gif/webp)
  • Logging con pino + pino-http
  • Health check endpoint GET /api/health

2.2 Modelos de Datos

  • Usuario: nombre, email, password (bcrypt 12 rounds), rol, resetToken
  • Alerta: tipo (7 categorias), descripcion, sector, estado, autor, fecha, lat/lng, adjuntos[], eliminado
  • Comentario: texto, autor, alerta, fecha

2.3 RBAC (Role-Based Access Control)

  • Roles: ciudadano, autoridad, administrador
  • Permisos centralizados en permisos.js (3 copias identicas)
  • Middleware: verificarToken, verificarPermiso, verificarCambioEstado
  • Ciudadano solo puede marcar sus alertas como "resuelto"
  • Admin/autoridad pueden cambiar cualquier estado
